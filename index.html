TermUI

A modern, declarative, and efficient Terminal UI library for Python.

üéØ Why TermUI?

Most terminal UI libraries fall into one of two extremes:

  • Low-level (like curses) ‚Äî powerful but verbose, every detail must be handled manually.
  • High-level frameworks (like textual) ‚Äî full-featured but heavy, with steep learning curves.

TermUI takes a different path:

  • Declarative, React-like API ‚Äî build UIs by composing widgets, not redrawing text.
  • Consistent by default ‚Äî centralized theming + variants mean your UI looks good with minimal effort.
  • Efficient and async ‚Äî a diff-based renderer ensures only what changes is updated.
  • Simple but powerful ‚Äî easy to start, flexible enough for full-screen apps.

The goal is to make terminal apps feel as natural to build as modern web apps, while staying fast, lean, and Pythonic.

‚ú® Core Principles

  • Declarative ‚Äì UIs are defined as nested components, not imperative drawing calls.
  • Themed ‚Äì colors and styles are centralized, ensuring consistency across widgets.
  • Composable ‚Äì small widgets combine to build complex layouts.
  • Efficient ‚Äì diff-based rendering avoids costly full-screen redraws.
  • Accessible ‚Äì easy to get started, but extensible for advanced users.

üöÄ Example

from termui import App, Button, Container, VerticalLayout

def on_click():
    print("Button clicked!")

app = App(
    VerticalLayout(
        Container(
            Button("Click Me", on_click=on_click, variant="primary")
        )
    )
)

app.run()

üé® Theming & Variants

TermUI ships with a familiar set of theme tokens:

primary, primary_content
secondary, secondary_content
accent, accent_content
neutral, neutral_content
base_100, base_200, base_300, base_content
info, info_content
success, success_content
warning, warning_content
error, error_content

Widgets map directly to these through variants:

Button("Save", variant="primary")
Button("Cancel", variant="secondary")
Button("Delete", variant="error")

‚ù§Ô∏è Philosophy

We believe terminal apps should be:

  • Consistent by design ‚Äî theming and variants give a polished look out of the box.
  • Efficient under the hood ‚Äî diff rendering ensures smooth performance.
  • Familiar to modern developers ‚Äî inspired by the best ideas from web UI libraries.

If building terminal apps feels like building for the web ‚Äî but simpler ‚Äî we‚Äôve done our job.
