Contributing to TermUI

🎉 Thanks for your interest in contributing to TermUI! Contributions of all kinds are welcome — from bug reports and documentation improvements to new widgets and features.

🛠️ Getting Started

  1. Fork the repository and clone your fork:
git clone https://github.com/<your-username>/termui.git
cd termui
  1. Set up a development environment with Poetry:
poetry install

This installs TermUI along with all dev dependencies (linters, formatters, testing tools).

  1. Activate the virtual environment:
poetry shell
  1. Run the test suite to make sure everything is working:
poetry run pytest
  1. Try out the examples:
poetry run python examples/button_demo.py

📐 Code Style

TermUI follows these conventions:

  • Python version: 3.9+
  • Linting: ruff
  • Formatting: black
  • Typing: mypy

Before committing, run:

poetry run ruff check .
poetry run black .
poetry run mypy termui
poetry run pytest

📝 Commit Guidelines

We follow the Conventional Commits specification. This makes the commit history readable and helps with automated changelogs.

Format:

<type>[optional scope]: <description>

Examples:

  • feat: add ProgressBar widget
  • fix(input): correct cursor position on delete
  • docs: update theming guide
  • refactor: simplify diffing algorithm

Common commit types:

  • feat – a new feature
  • fix – a bug fix
  • docs – documentation only changes
  • style – formatting, no logic changes
  • refactor – code change that neither fixes a bug nor adds a feature
  • test – adding or fixing tests
  • chore – build tasks, tooling, dependencies

🧩 Adding New Widgets

  1. Place your widget in termui/widgets/.

  2. Follow the existing API pattern:

  3. Use dataclass-style attributes for props (padding, align, variant).

  4. Support variants (primary, secondary, error, etc.) mapped to the theme.

  5. Add event hooks (on_click, on_change, etc.) where applicable.

  6. Write tests in tests/widgets/.

  7. Add an example under examples/.

🧪 Testing

We use pytest:

poetry run pytest -v

When adding features, please include tests that cover:

  • Basic rendering
  • Variant + theming behavior
  • Input handling (keyboard/mouse)

📝 Submitting Pull Requests

  1. Create a feature branch:
git checkout -b feat/my-new-widget
  1. Commit your changes with a Conventional Commit message:
git commit -m "feat: add RadioBox widget"
  1. Push to your fork:
git push origin feat/my-new-widget

  1. Open a Pull Request on GitHub:

  2. Describe what the change does.

  3. Link to any relevant issues.
  4. Add screenshots/gifs of widgets in action if possible.

🐛 Reporting Issues

If you find a bug, please open an issue and include:

  • What you expected to happen
  • What actually happened
  • Steps to reproduce
  • Your OS, Python version, and terminal emulator

❤️ Community

  • Be kind and respectful.
  • Small contributions matter — even fixing a typo helps!
  • If you're unsure about something, open a discussion.

Thanks again for helping make TermUI better 🚀
