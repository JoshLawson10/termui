termui.renderer

FrameBuffer 

FrameBuffer(width, height, background_color=None)

A double-buffered frame buffer for efficient terminal rendering.

The FrameBuffer maintains current and previous frame states to enable differential rendering, only updating characters that have changed.

Parameters:

Name Type Description Default

width

int

The width of the frame buffer in characters.

 required

height

int

The height of the frame buffer in characters.

 required

background_color

Optional[Color]

Optional background color for empty cells.

 None

background_color instance-attribute 

background_color = background_color

Background color of the frame buffer.

current_frame instance-attribute 

current_frame = [
    [(_get_empty_char()) for _ in (range(width))]
    for _ in (range(height))
]

The currently rendered frame.

dirty_regions instance-attribute 

dirty_regions = set()

Any regions of the screen marked as needing re-rendering.

height instance-attribute 

height = height

Width and height of the frame buffer.

inline instance-attribute 

inline = True

Whether to resize the terminal to the content size. Determined by 'screen.inline' in the renderer.

previous_frame instance-attribute 

previous_frame = [
    [(_get_empty_char()) for _ in (range(width))]
    for _ in (range(height))
]

The previous rendered frame.

clear 

clear()

Clear the current frame buffer to empty characters.

Name Type Description Default

 required

list[list[Char]]

A 2D list of Char objects representing the content.

bool

Whether to clip content to the frame buffer boundaries.

 True

get_rendered_output 

get_rendered_output()

Render only changed characters to the terminal.

Uses differential rendering to only update characters that have changed since the last frame, improving performance.

mark_entire_screen_dirty 

mark_entire_screen_dirty()

 required

Renderer 

Renderer(driver)

Main renderer that manages frame buffers and screen rendering.

The Renderer coordinates between the application, screens, and the terminal, handling frame buffering, screen changes, and terminal resizing.

dom_tree instance-attribute 

dom_tree = DOMTree()

The DOM tree representing the current screen's UI.

driver instance-attribute 

driver = driver

The terminal driver instance for rendering output.

frame_buffer instance-attribute 

frame_buffer = FrameBuffer(initial_width, initial_height)

The frame buffer instance for rendering the screen.

check_resize 

check_resize()

Returns:

Type Description
bool

True if the terminal was resized, False otherwise.

clear 

clear()

Name Type Description Default

screen

The screen to prepare for rendering.

 required

render 

render()

Render all widgets to the terminal.

Processes the DOM tree and renders all dirty widgets to the frame buffer, then outputs the changes to the terminal.
