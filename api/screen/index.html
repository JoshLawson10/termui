termui.screen

Screen 

Screen()

Bases: ABC

Base class for terminal UI screens.

A Screen represents a complete UI view that can contain widgets, handle input events, and manage its own lifecycle. Screens are mounted to an App instance and can have local keybinds.

active instance-attribute 

active = False

Whether the screen is currently mounted.

background_color instance-attribute 

background_color = None

Optional background color for the screen.

dom_tree instance-attribute 

dom_tree = DOMTree()

A DOM tree for managing widget hierarchies and rendering order.

inline instance-attribute 

inline = True

Whether the screen is inline (True) or should resize the terminal to fit (False).

is_active property 

is_active

Whether the screen is currently mounted.

local_keybinds property 

local_keybinds

Get the local keybinds for this screen.

Returns:

Type Description
list[Keybind]

A list of keybinds that are specific to this screen.

log property 

log

Expose the TermUI logger to the screen.

name instance-attribute 

name = ''

Name of the screen. Used by the application for identification.

build abstractmethod 

build()

Build and return the root layout for this screen.

This method should create and return the root layout containing all widgets for this screen. It's called when the screen is mounted and after terminal resizes.

Returns:

Type Description
Layout

The root layout widget that contains all screen content.

setup abstractmethod 

setup()

Set up the screen with initial configuration.

This method is called once when the screen is registered with the application. Use this for one-time initialization.

unmount()

update()

This method is called periodically by the application loop and can be used to update screen content, animations, or state based on time or other factors.
